ご要望に基づき、現在のプロジェクトで提案されている各シーンのノード構成を、指定の表記形式で提示します。

これらのシーンは、`RootScene.gd`によって生成・管理されます。

-----

## 1\. RootScene.tscn (アプリケーション管理者)

アプリケーションの永続的な管理者であり、画面全体の**状態遷移**を制御します。

  * *注: 以前の議論に基づき、ゲーム世界に不要な`World`と`Camera2D`ノードは省略しています。ルートノードは`Node`（または`Control`）が望ましいですが、ここでは既存の`Node2D`から変更します。*

<!-- end list -->

```text
root : Node2D : (root_scene.gd をアタッチ)
>> World : Node2D : (削除推奨ノード)
>> Camera2D : Camera2D : (削除推奨ノード)
> UI_Layer : CanvasLayer : 
>> UI_Holder : Control :  
>>> (ここに MainMenuUI や MissionSelectUI がロードされる)
>> SidebarToggle : TextureButton : 
>> (SidebarノードはGDScriptでインスタンス化してUI_Layerの子として追加される)
```

-----

## 2\. MainMenuUI.tscn (メインメニュー画面)

アプリ起動直後のハブとなる画面です。

```text
root : Control : (MainMenuUI.gd をアタッチ)
> PanelContainer : PanelContainer : (画面中央に配置)
>> VBoxContainer : VBoxContainer :
>>> TitleLabel : Label : (例: "Cyber Frontier")
>>> StartMissionButton : Button : ("ミッション開始")
>>> RankButton : Button : ("成績 / ランキング")
>>> OptionButton : Button : ("設定 / オプション")
>>> QuitButton : Button : ("アプリケーション終了")
```

-----

## 3\. MissionSelectUI.tscn (ミッション選択・詳細統合画面)

ミッションリストと詳細表示を左右に統合した画面です。

```text
root : Control : (MissionSelectUI.gd をアタッチ)
> PanelContainer : PanelContainer : (画面中央に配置)
>> VBoxContainer : VBoxContainer :
>>> TitleLabel : Label : ("ミッション選択")
>>> HSplitContainer : HSplitContainer : (リストと詳細を左右に分割)
>>>> ListContainer : VBoxContainer : (リスト格納用)
>>>>> ScrollContainer : ScrollContainer :
>>>>>> MissionListContainer : VBoxContainer : (ここにミッションボタンがコードで生成される)
>>>> DetailArea : VBoxContainer : (詳細表示エリア)
>>>>> DetailTitleLabel : Label : (選択中のミッション名)
>>>>> DetailDescText : RichTextLabel : (難易度、概要などの詳細)
>>>>> StartButton : Button : ("ミッションスタート")
>> BackButton : Button : ("メインメニューに戻る")
```

-----

## 4\. Sidebar.tscn (永続的なツールバー)

ミッション実行中に表示され、各種ツールのMDIウィンドウを起動します。

```text
root : Control : (sidebar.gd をアタッチ)
> SidebarContainer : Control : (アニメーションで左右にスライドするコンテナ)
>> VBoxContainer : VBoxContainer : (ツールボタンを縦に並べる)
>>> TerminalButton : TextureButton : (Terminal MDIを起動)
>>> NetworkMapButton : TextureButton : (Network Map MDIを起動)
>>> ... (その他のツールボタン)
>>> BackToMenuButton : Button : ("メインメニューへ戻る")
```

-----

## 5\. MDIWindow.tscn (動的ウィンドウテンプレート)

ミッション実行中に動的に生成される、各ツールの親ウィンドウです。

```text
root : Window : (mdi_window.gd をアタッチ)
> ContentContainer : Control : (ウィンドウの枠内にコンテンツを配置するためのコンテナ)
>> ContentUI : Control : (Placeholder: TerminalUI.tscn などがここにロードされる)
```

## 💻 3. ミッション実行画面 UI構成イメージ

この画面は、ミッションが開始された直後（`RootScene.start_mission_from_details`実行後）の状態です。`RootScene`がフルスクリーンUIを非表示にし、ゲームモードの要素を有効化します。

### 構成要素の動作

| 要素 | 動作 | 役割 |
| :--- | :--- | :--- |
| **背景 (RootScene)** | 固定 | MDIウィンドウ群の背景。黒や暗いテクスチャなどが想定されます。 |
| **サイドバー/トグル** | 表示/アニメーション | **ミッション遂行に必要なツール**（ターミナル、マップなど）を起動するためのアイコン群。 |
| **MDIウィンドウ** | 動的/自由配置 | ツール（ターミナルなど）の実際の操作画面。ユーザーが移動・リサイズ可能。 |
